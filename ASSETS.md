# Asset Library

The asset library is a collection of **self-contained sub-templates** that install individual Databricks artifacts: pipeline templates, ingestion jobs, pipelines, dashboards, etc. into any Databricks bundle.

Assets are installed via the Databricks CLI, one at a time, on demand. They do **not** require the core [databricks-bundle-template](README.md) and can be added to any existing Databricks bundle.

## Canonical install pattern

```bash
databricks bundle init https://github.com/vmariiechko/databricks-bundle-template \
  --template-dir assets/<asset-name>
```

Run the command from the root of your bundle project. The CLI will prompt for any asset-specific values and write files into your project relative to the current directory. Existing files are never overwritten. If the asset would collide with something in your project, the CLI errors cleanly.

## What an asset is not

- Not a modification of the core template. The main `databricks bundle init` flow is unaffected by assets.
- Not a patch to existing files. Assets only create new files. If an asset's integration requires a line change in `databricks.yml`, the asset's README states the exact one-liner.
- Not tied to the core template. Each asset is standalone and installs into any Databricks bundle project, not only ones generated by the main template in this repo.
